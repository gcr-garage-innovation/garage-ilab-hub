I-LAB 操作指南

面向第一次进入 Lab 的用户：快速了解工作站配置、模型清单、数据存储规则和问题处理方式。

工作站：GMK EVO-X2 内存：128GB 存储：2TB SSD 本地模型：Ollama
推荐顺序：先看硬件与模型，再按“快速开始”执行一次任务，最后按命名规则保存到 NAS。

1. 工作站配置

项目配置说明
设备型号GMK EVO-X2Lab 标准工作站
内存128GB支持多模型与中高复杂任务
存储2TB SSD本地运行 + 中间文件缓存
模型运行时Ollama本地模型调用入口

3. 快速开始（10分钟）

Step A：确认模型已就绪

【Windows 工作站本机，PowerShell】

ollama list

Step B：执行一次任务

【Windows 工作站本机，PowerShell】

ollama run qwen3:14b "请把这段会议纪要总结成3条行动项"

4. 数据存储（含 NAS）

数据类型保存位置说明
最终成果NAS 共享目录（建议）必须持久化，便于团队复用
中间文件本地临时目录不建议长期存放，离开前转存
模板与案例团队资源库统一版本管理

NAS 路径示例：\\NAS\shared\I-LAB\（请按实际环境替换）。

5. 文件命名规则（建议统一）

用途命名格式示例
项目产出项目名_场景_版本_YYYYMMDD_作者I-LAB_文档洞察_v1_20260716_Kyle
演示材料Demo_主题_对象_YYYYMMDDDemo_RAG_GarageTeam_20260716
数据文件Data_来源_用途_YYYYMMDDData_Survey_Training_20260716

建议使用英文/数字/下划线，避免空格和特殊字符。

6. 软件安装与权限

  • 临时工具：可申请后使用。
  • 系统级改动（驱动、服务、安全策略）：需管理员在维护窗口执行。
  • 未知来源工具：需评审，不可直接安装。
